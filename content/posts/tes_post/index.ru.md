# Прошивка и настройка OpenWrt на Cudy WR3000S

> **Модель:** Cudy WR3000S v1  
> **Серийник:** 2551xxxxx (новый флеш-чип → нужна прошивка OpenWrt 24.10.5+)  
> **Версия OpenWrt:** 25.12.2

---

## Что должно быть готово

- **Файл A** — промежуточная прошивка `WR3000S 1.0.zip` (скачан с Google Drive Cudy, распакован → внутри `.bin` файл)
- **Файл B** — официальный OpenWrt `openwrt-...-cudy_wr3000s-v1-squashfs-sysupgrade.bin` (скачан с firmware-selector.openwrt.org, версия 25.12.2)
- Компьютер подключён **Ethernet-кабелем** в LAN-порт роутера
- Wi-Fi на компьютере **выключен** (чтобы не было конфликта маршрутов)

---

## Часть 1. Прошивка

### Шаг 1 — Заливаем промежуточную прошивку через стоковый интерфейс Cudy

1. Открой браузер, зайди на **http://192.168.10.1**
2. Войди в веб-интерфейс Cudy (если роутер новый — пройди первичную настройку, можно с любыми параметрами, это неважно)
3. Перейди в **Advanced Settings → System → Firmware Upgrade**
4. Нажми кнопку выбора файла и выбери **Файл A** (распакованный `.bin` из архива `WR3000S 1.0.zip`)
5. Подтверди обновление
6. **Жди 3–5 минут.** Роутер перезагрузится. Не отключай питание и не отключай кабель!

### Шаг 2 — Входим во временный OpenWrt

После перезагрузки IP-адрес роутера изменится:

1. Если браузер не перенаправил автоматически — открой **http://192.168.1.1**
2. Логин: **root**
3. Пароль: **пустой** (просто нажми Enter / Login)

Ты попадёшь в интерфейс LuCI. Это промежуточная версия — она нужна только как трамплин для установки настоящего OpenWrt. **Ничего тут не настраивай.**

### Шаг 3 — Заливаем официальный OpenWrt

1. В LuCI перейди в **System → Backup / Flash Firmware**
2. Прокрути вниз до раздела **«Flash new firmware image»**
3. Нажми **«Flash image…»** и выбери **Файл B** (sysupgrade.bin 25.12.2)
4. ⚠️ **Сними галочку** «Keep settings and retain the current configuration»
5. Нажми **Continue**, затем подтверди прошивку
6. **Жди 3–5 минут.** Роутер снова перезагрузится

После перезагрузки снова зайди на **http://192.168.1.1** — теперь это чистый официальный OpenWrt 25.12.2.

## Часть 2. Базовая настройка OpenWrt

### Шаг 4 — Устанавливаем пароль root

1. [ ] Зайди на **http://192.168.1.1**
2. Войди как **root** без пароля
3. Перейди в **System → Administration**
4. В разделе **Router Password** задай надёжный пароль
5. Нажми **Save**

> С этого момента SSH-доступ тоже будет по этому паролю: `ssh root@192.168.1.1`

### Шаг 5 — Настраиваем интернет (WAN)

Если роутер подключён к провайдеру напрямую:

1. Воткни кабель от провайдера в **WAN-порт** роутера
2. Перейди в **Network → Interfaces**
3. Нажми **Edit** на интерфейсе **wan**
4. Выбери протокол:
    - **DHCP client** — если провайдер раздаёт IP автоматически (большинство случаев)
    - **PPPoE** — если провайдер дал логин/пароль (Ростелеком, Дом.ру и т.д.)
    - **Static address** — если провайдер дал фиксированный IP
5. Заполни нужные поля и нажми **Save & Apply**

Если роутер работает за другим роутером (второй роутер в цепочке):

1. Подключи кабель от основного роутера в WAN-порт
2. Протокол WAN оставь **DHCP client**
3. Убедись, что подсеть не пересекается (OpenWrt по умолчанию 192.168.1.0/24 — если основной роутер тоже на 192.168.1.x, смени LAN OpenWrt на другую, например 192.168.2.1)

Чтобы сменить LAN-адрес: **Network → Interfaces → lan → Edit → IPv4 address** → поменяй → **Save & Apply**. После этого зайди в интерфейс по новому адресу.

### Шаг 6 — Настраиваем Wi-Fi

1. Перейди в **Network → Wireless**
2. Ты увидишь два радиомодуля: **radio0** (2.4 GHz) и **radio1** (5 GHz)
3. Для каждого нажми **Edit**:
    - **ESSID** — имя сети (например: `MyNetwork`)
    - **Mode:** Access Point
    - Вкладка **Wireless Security**:
        - **Encryption:** WPA2-PSK (или WPA3-SAE если все устройства поддерживают)
        - **Key:** пароль Wi-Fi
4. Нажми **Save & Apply**
5. Если радиомодули выключены — нажми **Enable** рядом с каждым



### Шаг 8 — Обновляем пакеты и ставим полезное

Это все не обязательно. 

Подключись по SSH:

```bash
ssh root@192.168.1.1
```

Обнови списки пакетов:

```bash
apk update
```

Установи веб-интерфейс LuCI с поддержкой HTTPS (если ещё не установлен):

```bash
apk add luci luci-ssl luci-i18n-base-ru
/etc/init.d/uhttpd restart
```

Другие полезные пакеты:

```bash
# Русский язык для LuCI (если не поставился выше)
apk add luci-i18n-base-ru

# Утилиты для диагностики
apk add tcpdump curl wget

# Поддержка USB (если нужно подключать USB-накопители)
apk add kmod-usb-storage block-mount
```

## Часть 3. Установка Podkop (для обхода блокировок)

### Шаг 9 — Устанавливаем Podkop

```bash
# Скачиваем и запускаем установщик Podkop
wget -O /tmp/install_podkop.sh https://raw.githubusercontent.com/itdoginfo/podkop/main/install.sh
sh /tmp/install_podkop.sh
```

После установки в LuCI появится раздел **Services → Podkop**.

### Шаг 10 — Настраиваем Podkop с VLESS+Reality+xHTTP

1. В LuCI перейди в **Services → Podkop**
2. Добавь новый профиль прокси
3. Вставь свою VLESS-ссылку (формат `vless://...`)
4. Выбери режим маршрутизации:
    - **Список доменов/IP** — только указанные ресурсы идут через VPN
    - **Весь трафик** — всё через VPN
5. Для списков заблокированных ресурсов можно подключить готовые списки (antifilter, community-списки)
6. **Save & Apply**


### Шаг 11 — Проверяем работоспособность

```bash
# Проверяем, что Podkop запущен
/etc/init.d/podkop status

# Проверяем доступность через VPN
curl -s https://ifconfig.me
# Должен показать IP твоего VPS, а не провайдера

# Проверяем DNS
nslookup google.com
```

## Полезные команды для обслуживания

```bash
# Обновление всех пакетов
apk update && apk upgrade

# Бэкап конфигурации (перед экспериментами)
sysupgrade -b /tmp/backup.tar.gz
# Скачать бэкап: через LuCI → System → Backup / Flash Firmware → Generate archive

# Просмотр логов
logread
logread -f  # в реальном времени

# Перезапуск сети


# Перезапуск Wi-Fi
wifi down && wifi up

# Информация о системе
cat /etc/openwrt_release
free -m
df -h
```

## Восстановление на стоковую прошивку Cudy

Если что-то пошло не так и нужно вернуться:

1. Зайди на https://www.cudy.com/blogs/faq/how-to-recovery-the-cudy-router-from-openwrt-firmw
2. Скачай стоковую прошивку для WR3000S
3. Через LuCI: **System → Backup / Flash Firmware → Flash image** → загрузи стоковую прошивку (сними галочку «Keep settings»)
4. Если LuCI недоступен — используй TFTP-восстановление (инструкция на той же странице Cudy)

### Важные ньюансы

1) Podkop может не принимать ссылку на конфиг подключения vless. Поэтому ее нужно конвертировать в подходящий формат с помощью утилиты https://github.com/EikeiDev/vless-xtls-converter
2) Обычный singbox не подойдет для работы с xhttpx, поэтому нужно использовать https://github.com/EikeiDev/OpenWRT-sing-box-extended
3) UDP Может работать нестабильно, поэтому лучше попробовать TLS.![[Pasted image 20260407201902.png]]
4) Браузер может не допускать на нужные ресурсы, как будто трафик идет не через vpn. Нужно Выключить в настройка "Безопасный DNS". Т.к с этой настройкой трафик может миновать перехват роутером и идти напрямую в гугл и дальше.
5) Не все сервисы есть в публичных списка, свои можно добавлять по необходимости![[Pasted image 20260407201948.png]]

# grpc конфиг

XHTTP на текущий момент может работать нестабильно. Я пропробую создать grpc vless reality чтобы singbox работал без перебоев и терял dns.
В настройках подком он принимает в формате url, конвертация в другой формат не нужна.

### Пример настройки конфика для grpc vless:

**Заходи в 3X-UI → Inbounds (Подключения) → Add Inbound (Добавить подключение)**

**Основные настройки (верхний блок):**

|Поле|Значение|Пояснение|
|---|---|---|
|**Remark**|`router-grpc`|Любое имя, чтобы отличать от xHTTP inbound|
|**Protocol**|`vless`|Протокол|
|**Listening IP**|_пусто_|Слушать на всех интерфейсах|
|**Port**|`2053`|Не 443 — он уже занят xHTTP. Подойдёт любой свободный: 2053, 8443, 2083 и т.д.|
|**Transmission / Network**|`grpc`|Ключевой момент — выбираем gRPC транспорт|

**Блок Client (клиент):**

|Поле|Значение|Пояснение|
|---|---|---|
|**Email**|`router`|Имя клиента, любое|
|**Flow**|_пусто_|Для gRPC flow НЕ используется. Если стоит `xtls-rprx-vision` — **убери**. Vision работает только с TCP|

**Блок gRPC Settings:**

|Поле|Значение|Пояснение|
|---|---|---|
|**Service Name**|`grpc-tunnel`|Секретный путь, придумай своё. Без слеша в начале. Это аналог path в xHTTP|

**Блок Security (безопасность):**

| Поле                   | Значение             | Пояснение                                                                                                                                                                                       |
| ---------------------- | -------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Security**           | `reality`            | Обязательно Reality                                                                                                                                                                             |
| **uTLS**               | `chrome`             | Маскировка под Chrome                                                                                                                                                                           |
| **Dest (Target)**      | `ozon.ru:443`        | Тот же сайт-донор что и у xHTTP, или другой. Главное — незаблокированный зарубежный или крупный российский сайт с TLS 1.3, H2, без CDN                                                          |
| **SNI (Server Names)** | `ozon.ru`            | Совпадает с Dest                                                                                                                                                                                |
| **ALPN**               | _пусто_              | Для gRPC оставить пустым                                                                                                                                                                        |
| **Get New Cert**       | **Нажми эту кнопку** | Генерирует пару ключей (Public Key + Private Key). Если хочешь использовать **те же ключи** что и у xHTTP inbound — вместо генерации скопируй Private Key и Public Key из существующего inbound |
| **Sniffing**           | ✅ **Включить**       | Определение типа трафика                                                                                                                                                                        |
